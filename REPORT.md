# 重構報告

## 這次重構做了什麼

這次不是在舊版專案上小修小補，而是直接把產品方向切成「輕量化多 agent 會議室」。

重構後的核心原則：

- 單機優先
- 一個畫面、多個 agent
- 會議流程盡量簡單
- 會議中只保留暫存記憶
- 會後匯出文字或 Python 檔案，再進入長期記憶
- OpenClaw 只作為整合對象，不重做它的 Gateway / Skills

## 已完成項目

### 1. 規格重寫

- 已重寫 [agent.md](C:\codex\agent-meeting-room\agent.md:1)
- 將產品定位改為本地優先的多 agent 會議室
- 明確定義 MVP 不做重 PM 流程

### 2. 資料模型重建

新版資料模型改成只保留會議室真正需要的幾個實體：

- `AppSetting`
- `RoleProfile`
- `Meeting`
- `MeetingParticipant`
- `MeetingMessage`
- `MemoryArchive`

這樣的好處是資料結構比較扁平，維護成本低，也更適合單機先跑穩。

### 3. 後端 API 重做

後端改成圍繞以下能力：

- 讀取與更新設定
- 管理角色
- 建立會議
- 進行討論回合
- 關閉會議
- 匯出會議結果
- 查看長期記憶列表

同時保留 `mock` 與 `openai_compatible` 兩種模型模式，方便本地測試和未來串接真實模型。

### 4. 前端介面重做

前端改成單頁會議室介面，重點集中在：

- 中央討論區
- 右側 agent 狀態與記憶區
- 底部輸入區
- 設定抽屜
- 建立會議視窗

這版先優先確保功能完整與穩定，視覺細節之後再繼續優化。

### 5. 記憶與匯出流程建立

已完成：

- 會議中暫存記憶更新
- 回合摘要寫入暫存記憶
- 匯出 `text`
- 匯出 `python`
- 匯出後建立 `MemoryArchive`
- 檔案寫入 `exports/`

### 6. 測試重寫

已改成針對新版產品的測試：

- 首頁渲染
- bootstrap 初始化
- 建立會議並跑一輪
- 匯出與記憶存檔

## 修正過的關鍵問題

在重構過程中已經處理兩個會直接影響結果的 bug：

- SQLAlchemy relationship cache 造成回傳會議資料時看不到最新訊息
- JSON 欄位原地修改沒有被正確持久化，導致 `temporary_memory` 更新失敗

目前這兩個問題都已修正。

## 目前適合的使用方式

這版最適合：

- 本地單機開會
- 驗證多角色討論流程
- 測試角色提示詞
- 測試不同 OpenAI compatible 模型
- 匯出討論成果做二次處理

## 暫時不做的事

以下內容這版刻意不做，避免系統變重：

- 複雜 PM 管線
- 任務拆解工作流引擎
- 多租戶權限
- 重型長期記憶檢索系統
- 重做 OpenClaw Gateway
- 重做 OpenClaw Skills 系統

## 下一步建議

下一個迭代最值得做的方向有三個：

1. 優化前端閱讀性，讓主視覺更像真的會議室
2. 加入 OpenClaw agent 匯入或對接流程
3. 設計更清楚的角色模板與角色設定體驗
